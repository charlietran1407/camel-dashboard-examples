# LangChain4j & Apache Camel Integration Reference Notes

These reference notes document the behavior and integration patterns for Apache Camel's LangChain4j components (`camel-langchain4j-embeddings` and `camel-langchain4j-embeddingstore`), along with diagnostic insights on similarity thresholds.

---

## 1. Component Data Flows & Message Mapping

### A. `camel-langchain4j-embeddings`
When a route directs a message body (usually `String`) to the `langchain4j-embeddings` component:
* **Message Body:** The original `String` text body is **preserved** and **not** replaced by the vector representation.
* **Headers Produced:**
  * `CamelLangChain4jEmbeddingsVector`: Holds the dense vector embedding as a primitive float array (`float[]`).
  * `CamelLangChain4jEmbeddingsTextSegment`: Holds the `dev.langchain4j.data.segment.TextSegment` object wrapping the input text and metadata.
  * `CamelLangChain4jEmbeddingsFinishReason`: Holds the finish reason of the model call.

### B. `camel-langchain4j-embeddingstore`
The embedding store component coordinates with the headers generated by the embeddings component:

* **`ADD` Action:**
  * Automatically reads the embedding vector from the `CamelLangChain4jEmbeddingsVector` header.
  * Reads the associated text segment from the `CamelLangChain4jEmbeddingsTextSegment` header (or falls back to the message body if headers are absent).
  * Stores both the vector and text segment into the target database (e.g., `InMemoryEmbeddingStore`).

* **`SEARCH` Action:**
  * Reads the query vector from the `CamelLangChain4jEmbeddingsVector` header.
  * **`returnTextContent: true`** (Default in many RAG examples): Maps the search result of `List<EmbeddingMatch<TextSegment>>` to `List<String>`, returning only the raw text segments in the message body.
  * **`returnTextContent: false`**: Preserves the full `List<EmbeddingMatch<TextSegment>>` in the message body.

---

## 2. Serialization Gotchas

### Jackson `FAIL_ON_EMPTY_BEANS`
When `returnTextContent` is set to `false`, the search route returns `List<EmbeddingMatch<TextSegment>>`. Marshalling this list directly to JSON via Camel's `marshal().json()` fails because `EmbeddingMatch` does not expose standard bean getters, causing a Jackson `InvalidDefinitionException`.

### Workaround (Groovy Mapper):
Use a Groovy expression step to map the results list to simple maps before marshalling:
```yaml
- setBody:
    expression:
      groovy:
        expression: |
          def results = []
          request.body.each { match ->
              results << [
                  score: match.score(),
                  text: match.embedded().text()
              ]
          }
          return results
- marshal:
    json: {}
```

# Tài liệu tham khảo tích hợp LangChain4j & Apache Camel (Tiếng Việt)

Tài liệu này ghi lại hành vi và các mẫu tích hợp của các component LangChain4j trong Apache Camel (`camel-langchain4j-embeddings` và `camel-langchain4j-embeddingstore`), kèm theo phân tích chi tiết về ngưỡng tương đồng vector.

---

## 1. Luồng dữ liệu & Mapping Message

### A. `camel-langchain4j-embeddings`
Khi route gửi message body (thường là `String`) đến endpoint `langchain4j-embeddings`:
* **Message Body:** Body gốc kiểu `String` được **giữ nguyên**, không bị ghi đè bởi vector.
* **Headers sinh ra:**
  * `CamelLangChain4jEmbeddingsVector`: Chứa vector embedding dưới dạng mảng float (`float[]`).
  * `CamelLangChain4jEmbeddingsTextSegment`: Chứa đối tượng `dev.langchain4j.data.segment.TextSegment` bọc text và metadata.
  * `CamelLangChain4jEmbeddingsFinishReason`: Chứa lý do kết thúc cuộc gọi model.

### B. `camel-langchain4j-embeddingstore`
Component này phối hợp trực tiếp với các header do component embeddings tạo ra:

* **Action `ADD`:**
  * Tự động lấy query vector từ header `CamelLangChain4jEmbeddingsVector`.
  * Lấy text segment từ header `CamelLangChain4jEmbeddingsTextSegment` (hoặc body nếu thiếu header).
  * Lưu trữ cả vector và text segment vào cơ sở dữ liệu vector mục tiêu (như `InMemoryEmbeddingStore`).

* **Action `SEARCH`:**
  * Đọc query vector từ header `CamelLangChain4jEmbeddingsVector`.
  * **`returnTextContent: true`** (Mặc định trong nhiều ví dụ RAG): Chuyển đổi kết quả tìm kiếm từ dạng danh sách `List<EmbeddingMatch<TextSegment>>` sang `List<String>` chứa text thuần túy trong body.
  * **`returnTextContent: false`**: Giữ nguyên danh sách `List<EmbeddingMatch<TextSegment>>` trong body.

---

## 2. Lưu ý về Tuần tự hóa (Serialization)

### Jackson `FAIL_ON_EMPTY_BEANS`
Khi chuyển `returnTextContent` sang `false`, route search sẽ trả về `List<EmbeddingMatch<TextSegment>>`. Việc marshal trực tiếp list này ra JSON bằng `.json()` của Camel sẽ bị lỗi vì đối tượng `EmbeddingMatch` không chứa các getter chuẩn theo mô hình Java Bean.

### Giải pháp (Groovy Mapper):
Sử dụng Groovy expression để ánh xạ list kết quả thành danh sách các Map đơn giản trước khi chuyển sang JSON:
```yaml
- setBody:
    expression:
      groovy:
        expression: |
          def results = []
          request.body.each { match ->
              results << [
                  score: match.score(),
                  text: match.embedded().text()
              ]
          }
          return results
- marshal:
    json: {}
```
